    1.双向链表和list

    双向链表

    结构：双向链表结构中元素在内存中不是紧邻空间，而是每个元素中存放上一个元素和后一个元素的地址

    • 第一个元素称为（头）元素，前连接（前置指针域）为nil
    • 最后一个元素称为 尾（foot）元素，后连接（后置指针域）尾nil

    优点：

    1. 在执行新增元素或删除元素时效率高，获取任意一个元素，可以方便的在这个元素前后插入元素
    2. 充分利用内存空间，实现内存灵活管理可实现正序和逆序遍历
    3. 头元素和尾元素新增或删除时效率较高

    缺点：

    1. 链表增加了元素的指针域，空间开销比较大
    2. 遍历时跳跃性查找内容，大量数据遍历性能低

    双向链表容器 List

    在Go语言标准库的container/list包提供了双向链表Lis
    tList结构体定义如下：

    • root表示根元素
    • len表示链表中有多少元素

    其中Element结构体定义如下

    • next表示下一个元素，使用Next()可以获取到
    • prev表示上一个元素，使用Prev()可以获取到
    • list表示元素属于哪个链表
    • Value表示元素的值，interface()在Go语言中表示任意类型

    双向循环链表 Ring

    循环链表特点是没有节点的指针域为nil，通过任何一个元素都可以找到其它元素。

    双向循环链表和双向链表区别：

    • 双向循环链表没有严格意义上的头元素和尾元素
    • 没有元素的前连接和后连接为nil
    • 一个长度为n的双向循环链表，通过某个元素向某个方向移动，在查找最多n-1次，一定会找到另一个元素

    2. Go&&Redis分布式锁

    分布式系统中共享同一个资源时，就需要分布式锁来确保变更资源的一致性。

    应用场景

    1. 用户下单需要锁住 uid，防止用户重复下单。
    2. 用在库存扣减上，锁住库存，可以防止库存超卖。
    3. 用在余额扣减场景，锁住账户，防止并发操作。

    分布式锁特征

    1. 排他性：锁只能被第一个持有者拥有。
    2. 防死锁：超时自动释放
    3. 可重入：锁持有者支持可重入，注意防止重入时超时释放
    4. 高可用

    Redis set

    • EX：键过期时间/秒
    • PX：键过期时间/毫秒
    • NX：键不存在时，才进行设置。
    • XX：键已经存在时，才进行设置。

    3. 正则

    Go中正则表达式采用RE2语法

    基础规则

    字符

    • . 匹配任意字符 abc. 匹配 abcd,abcx,abc9
    • [] 匹配任一字符 [abc]d 匹配 ad,bd,cd
    • [-] 匹配范围 [a-zA-Z]
    • [^] 匹配除括号中的任意字符 [^xy]b 匹配除xb,yb以外

    数量限定

    • ? 前面单元匹配0/1次
      • 前面单元匹配1次或多次
    • * 前面单元匹配任意次
    • {1,2} 匹配上下限次数（如1-2次）

    特殊字符

    • \ 转义
    • | 或 [a|b] 匹配 a或b
    • () 组成单元

    6.布隆过滤器

    布隆过滤器（Bloom Filter）是一种在大规模数据集合中快速检索数据的概率算法。它利用位数组和多个哈希函数来判断一个元素是否存在于集合中，可以高效地在海量数据中判断出某个元素是否出现过。

    应用场景

    1. 数据库查询优化： 在某些情况下，为了避免不必要的数据库查询，可以使用布隆过滤器来判断一个查询是否有可能存在于数据库中。如果查询不存在于过滤器中，则可以直接返回结果而无需执行数据库查询。
    2. 缓存穿透缓解： 如果缓存中没有所需数据，客户端请求将通过到达后端服务或数据库。当攻击者大量请求不存在于系统中的数据时，会导致缓存和数据库的负载增加。使用布隆过滤器可以快速判断该请求是否合法并拒绝非法请求，从而减轻服务器压力。
    3. URL 访问过滤： 在某些情况下，需要对不同来源的 URL 进行访问控制。使用布隆过滤器，可以快速判断一个 URL 是否被允许访问，从而有效地防止未经授权的访问。

    注意：如果 Bloom Filter 内部有太多的哈希函数或者位数组大小过小，会增加误判率。因此，在实践中，通常需要根据实际场景不断调整哈希函数数量和位数组大小来平衡误判率和存储空间消耗。

    7. PGO

    PGO是一种编译器优化技术，可以通过对程序的运行情况进行分析，优化生成的代码，从而提高程序性能。

    使用方式

    GO中PGO是使用一个profile文件指导编译器生成更优化的代码。

    1. 生成profile文件：在程序运行时记录程序统计数据，如函数调用次数、代码分支、循环等。
    2. 编译程序
    3. 重新运行程序：使用新生成的二进制文件，再次运行程序

    效果：PGO应用比未使用PGO时提高5%~20%

    场景

    1. Web应用：提高响应速度和吞吐量
    2. 数据库应用：优化SQL查询和数据库访问代码，提高查询性能。
    3. 游戏应用：优化游戏逻辑和图形渲染代码，提高帧率和流畅度。
    4. 操作系统内核：提高操作系统的响应速度和稳定性。
