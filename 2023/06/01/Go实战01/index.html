    基础

    1.基础数据类型

    整型

    整型分为两个大类：按长度分为：int8、int16、int32、int64 对应的无符号整型：uint8、uint16、uint32、uint64

    特殊整型：uint、int，在32位系统中就是uint32/int32，64位系统中就是uint64/int64

    注意事项：

    • 获取对象的长度的内建len()函数返回的长度可以根据不同平台的字节长度进行变化。
    • 切片或 map 的元素数量等都可以用int来表示。
    • 在涉及到二进制传输、读写文件的结构描述时，为了保持文件的结构不会受到不同编译目标平台字节长度的影响，不要使用int和 uint。
    数字字面量（Number literals syntax）
    1. Go1.13版本之后引入了数字字面量语法，这样便于开发者以二进制、八进制或十六进制浮点数的格式定义数字，例如：
    • v := 0b00101101， 代表二进制的 101101，相当于十进制的 45。
    • v := 0o377，代表八进制的 377，相当于十进制的 255。
    • v := 0x1p-3，代表十六进制的 1 除以 2³，也就是 0.125。

    注意：
    8进制以0开头：var b int = 077
    16进制以0x开头：var c int = 0xff

    1. 允许用 _ 来分隔数字，比如说： v := 123_456 表示 v 的值等于 123456。

    布尔型

    Go语言中以bool类型进行声明布尔型数据，布尔型数据只有true（真）和false（假）两个值。

    注意：

    • 布尔类型变量的默认值为false
    • 不允许将整型强制转换为布尔型
    • 布尔型无法参与数值运算，也无法与其他类型进行转换。

    字符byte和rune

    字符用单引号包裹起来，如：var a = ‘中’

    • uint8类型，或者叫 byte 型，代表了ASCII码的一个字符。
    • rune类型，代表一个 UTF-8字符。rune类型实际是一个int32，用来处理 Unicode（中文、日文或者其他复合字符）

    解析：

    • 字符串底层是一个byte数组，所以可以和[]byte类型相互转换。
    • 字符串是不能修改的
    • 字符串是由byte字节组成，所以字符串的长度是byte字节的长度。
    • rune类型用来表示utf8字符，一个rune字符由一个或多个byte组成。

    修改字符串：
    要修改字符串，需要先将其转换成[]rune或[]byte，完成后再转换为string。无论哪种转换，都会重新分配内存，并复制字节数组。

    2.指针详解

    指针是一个代表着某个内存地址的值, 这个内存地址往往是在内存中存储的另一个变量的值的起始位置

    指针特性：:

    • 默认值: nil.
    • 操作符 & 取变量地址, * 通过指针访问目标对象.
    • 不支持指针运算, 不支持 -> 运算符, 直接用 . 访问目标成员.

    指针地址和变量空间

    指针就是地址, 指针变量就是存储地址的变量：

    栈帧

    stack（栈）heap（堆）

    栈帧: 用来给函数运行提供内存空间, 取内存于 stack（栈） 上。
    生命周期：当函数调用时, 产生栈帧; 函数调用结束, 释放栈帧。
    存储内容：局部变量、形参、内存字段描述值
    备注：形参与局部变量存储地位等同

    空指针与野指针

    空指针: 未被初始化的指针.

    野指针: 被一片无效的地址空间初始化.

    指针变量的内存存储

    表达式 new(T) 将创建一个 T 类型的匿名变量，为 T 类型的新值分配并清零一块内存空间，然后将这块内存空间的地址作为结果返回；
    这个结果就是指向这个新的 T 类型值的指针值, 返回的指针类型为 *T

    new() 创建的内存空间位于heap上, 空间的默认值为数据类型的默认值. 如: p := new(int) 则 *p 为 0

    这时 p 就不再是空指针或者野指针
    我们只需使用 new() 函数, 无需担心其内存的生命周期或者怎样将其删除, 因为Go语言的内存管理系统会帮我们打理一切.

    左值右值

    问题：y int 和 x = y中的y相同吗？答：不相同

    • var y int = 20 中的 y 代表的是内存空间, 我们一般把这样的称之为左值;
    • x = y 中的 y 代表的是内存空间中的内容, 我们一般称之为右值
    • x = y 表示的是把 y 对应的内存空间的内容写到x内存空间中
    • 等号左边的变量代表变量所指向的内存空间, 相当于写操作
    • 等号右边的变量代表变量内存空间存储的数据值, 相当于读操作

    函数的传参

    函数传参永远都是值传递：

    • 传地址(引用)：将地址值作为函数参数传递
    • 传值(数据)： 将实参的值拷贝一份给形参

    传值的思想是: 在自己的栈帧空间中修改自己栈帧空间中的值.

    defer语句

    特性：

    1. 在函数退出时去执行
    2. defer语句经常使用成对的操作，比如打开和关闭，连接和断开，加锁和解锁

    用途：

    1. 当函数返回时，执行defer语句，因此可以用来做资源清理
    2. 多个defer语句，遵循栈的特性，按先进后出的方式执行
    3. defer语句中的变量，在defer声明时就决定了

    预计算参数：
    调用defer关键字，会立刻拷贝函数中引用的外部参数

    优化：传入匿名函数

    执行机制：1.14后，defer在函数体中最多执行一次时，将结构体分配到栈上并调用

    内置函数

    1. close：主要用来关闭channel
    2. len：用来求长度，比如string、 array、 slice、 map、 channel
    3. new：用来分配内存，主要用来分配值类型，比如int、 struct。返回的是指针
    4. make：用来分配内存，主要用来分配引用类型，比如chan、 map、 slice
    5. append：用来追加元素到数组、 slice中
    6. panic和recover：用来做错误处理

    变量作用域

    全局变量：在函数外面的变量，被称为全局变量，该变量对整个程序都生效。
    局部变量：
    1）函数内定义：仅仅在变量所定义的函数内生效
    2）语句块内定义：仅仅在变量所定义的语句块内生效

    可见性：

    • 变量首字母大写是可导出的能够被其他包访问或调用。
    • 变量小写表示是私有的，不能被外部的包访问。

    闭包

    闭包是由函数及其相关引用环境组合而成的实体(即：闭包=函数+引用环境)。

    函数f返回了一个函数，返回的这个函数就是一个闭包。
    这个函数中本身是没有定义变量i的，而是引用了它所在的环境（函数f）中的变量i，这个变量i就和匿名函数打包为一个整体，在闭包的这个有效区间内一直是生效存在的。

    备注：
    x 会累加的原因是因为它被定义在闭包中，而闭包中的函数可以访问和修改其外部环境中的变量。
    闭包函数不会重新创建和初始化变量 x，因此使得 x 的值可以在多次调用闭包函数之间保持连续，并在每次操作中记录下来。

    闭包的延迟绑定：
    闭包保存/记录了它产生时的外部函数的所有环境，执行这个闭包的时候，才会去外部环境寻找最新的数值。

    切片 slice

    切片是长度可变、容量固定的相同的元素序列。Go语言的切片本质是一个数组。

    • 容量固定是因为数组的长度是固定的，切片的容量即隐藏数组的长度。

    1.创建切片：
    1）make ( []Type ,length, capacity )
    2) make ( []Type, length)
    3) []Type{}
    4) []Type{value1 , value2 , … , valueN }

    2.切片与隐藏数组：
    一个切片是一个隐藏数组的引用，并且对于该切片的切片也引用同一个数组。

    解析：切片slice0 、 slice1 和 slice2是同一个底层数组的引用，所以slice2改变了，其他两个都会变。

    1. 追加、复制切片：
      追加、复制切片，用的是内置函数append和copy，copy函数返回的是最后所复制的元素的数量。 slice的长度为：0,slice为：[]

    4.append
    append可以向一个切片后追加一个或多个同类型的其他值。
    如果追加的元素数量超过了原切片容量，那么最后返回的是一个全新数组中的全新切片。
    如果没有超过，那么最后返回的是原数组中的全新切片。
    无论如何，append对原切片无任何影响。

    slice为：[1 2 3 4 5 6]
    操作的切片：[1 2]
    slice为：[1 2 50 60 5 6]
    操作的切片：[1 2]

    6. make和new

    new 关键字只分配内存，当我们想要获取指向某个类型的指针时可以使用 new ；
    make 关键字的主要作用是初始化内置的数据结构，也就是 slice、map 和 channel 的初始化。

    new

    1. new 函数只接受一个参数，这个参数是一个类型
    2. 并且返回一个指向该类型内存地址的指针。
    3. new 函数会把分配的内存置为零，也就是类型的零值。

    make

    1. make只用于 chan、map 以及 slice 的内存创建
    2. 返回的类型是这三个类型本身（都是引用类型）。
    3. make分配空间后，会进行初始化。

    同样，在使用make函数初始化映射或通道时，也会分配相应的空间并初始化其内部状态。

    结构体比较

    1. 结构体只能比较是否相等，但是不能比较大小。
    2. 相同类型的结构体才能够进行比较，结构体是否相同不但与属性类型有关，还与属性顺序相关
    3. 如果 struct 的所有成员都可以比较，则该 struct 就可以通过 == 或 != 进行比较是否相等，比较时逐个项进行比较，如果每一项都相等，则两个结构体才相等，否则不相等；

    可比较类型：bool、数值型、字符、指针、数组等
    不可比较类型：切片、map、函数
