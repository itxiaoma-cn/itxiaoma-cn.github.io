    服务容器和服务提供者

    服务容器 ServiceContainner

    ServiceContainner通过依赖注入将ServiceProvider提供的能力注册进服务容器内部。

    Laravel直接通过容器的实例化对象，找到对应的服务，就可以直接使用其提供的能力了。

    ServiceContainner就是一个类的实例化对象，启动时加载所有可用服务，用的时候再解析调用其中的方法。

    总结

    生成服务容器的过程：

    1.通过instance和register方法注册到instances数组
    2.通过singleton和bind方法绑定到bindings数组

    app.php基本流程

    1. 实例化app (new Illuminate\Foundation\Application())
      1.1 运行构造函数 __construct()
       1.1.1 设置基础路径 `$this->setBasePath($basePath);`
 1.1.2 创建并绑定基础服务容器（app,Container） `$this->registerBaseBindings();`
 1.1.3 注册基础服务提供者 ​`$this->registerBaseServiceProviders();`
 1.1.3 注册基础服务别名`$this->registerCoreContainerAliases();`
    2. 把App\Http\Kernel::class类下的参数映射到lluminate\Contracts\Http\Kernel::class类中
    3. 把App\Console\Kernel::class类下的参数映射到Illuminate\Contracts\Console\Kernel::class类中
    4. 把App\Exceptions\Handler::class类下的参数映射到Illuminate\Contracts\Debug\ExceptionHandler::class类中
    5. 返回app资源 return $app;
