    相关概念

    架构组件

    Master node：主控节点

    • API Server：集群统一入口（对外），以restful方式交给etcd存储
    • controller-manager：处理集群中常规后台任务，一个资源对应一个控制器
    • scheduler：节点调度，选择node节点应用部署
    • etcd：存储系统，用于保存集群相关的数据

    Worker node：工作节点

    • kubelet：管理本机容器
    • kube-proxy：提供网络

    核心概念

    通过Service定义访问规则，通过Controller部署POD

    1. POD

    • 最小部署单元
    • 一组容器的集合
    • POD间共享网络
    • 生命周期短暂

    2. Controller

    • 确保预期的POD副本数量
    • 有状态应用部署：有特定使用条件（依赖存储/网络IP）
    • 无状态应用部署：直接可用
    • 确保所有node运行同一个POD
    • 一次性任务和定时任务

    3.Service

    • 定义一组POD的访问规则

    Kubeadm部署

    Kubeadm是一个K8s部署工具，提供kubeadm init和kubeadm join，用于快速部署K8s集群

    环境配置

    系统版本：Centos7

    关闭防火墙

    systemctl stop firewalld
systemctl disable firewalld

    关闭selinux

    # 永久
sed -i 's/enforcing/disabled/' /etc/selinux/config 

# 临时
setenforce 0

    关闭 swap

    # 临时
swapoff -a  

# 永久
sed -ri 's/.*swap.*/#&/' /etc/fstab

    重置主机名

    hostnamectl set-hostname k8smaster

    配置hosts

    cat >> /etc/hosts << EOF 
10.206.0.15 k8s-master 
10.206.0.17 k8s-node1 
10.206.0.10 k8s-node2 
EOF

    配置网络

    cat > /etc/sysctl.d/k8s.conf << EOF 
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1 
EOF

# 生效
sysctl --system

    部署Kubenetes Master

    在Master节点执行

    kubeadm init --apiserver-advertise-address=10.206.0.15 --image-repository registry.aliyuncs.com/google_containers --kubernetes-version v1.18.0 --service-cidr=10.96.0.0/12 --pod-network-cidr=10.244.0.0/16

    得到token

    kubeadm join 10.206.0.15:6443 --token iv8baz.f2yagtk257ilmanr \
    --discovery-token-ca-cert-hash sha256:b43a11c9feeab057ee3d6ee91fd7e96dfc75859911f96ff1e89e9578d0801c23

cat >> /etc/hosts << EOF
124.220.21.78 master 
106.54.14.202 node1
EOF
