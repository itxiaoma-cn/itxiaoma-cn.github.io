    • 4.配置文件

    官网下载：http://download.redis.io/redis-stable/redis.conf

    Units 单位

    配置大小单位，只支持bytes，不支持bit

    大小写不敏感

    INCLUDES 包含

    提取公共配置文件

    ####NETWORK 网络

    配置支持原创连接
    #bind 127.0.0.1 -::1 //注释绑定本地连接
protected-mode no         //关闭保护模式

    端口号：默认6379

    port 6379

    GENERAL 通用

    开启后台启动
    daemonize yes
    进程号的路径
    pidfile /var/run/redis_6379.pid
    日志级别
    loglevel notice
    日志输出路径
    logfile ""
    数据库数数量
    databases 16

    SECURITY 安全

    开启访问密码
    requirepass foobared

    其他

    最大客户端连接数
    maxclients 10000
    设置最大内存
    maxmemory <bytes>
    内存超出上限：按规则移除
    maxmemory-policy noeviction

    注：如果redis无法按规则移除，或者设置了“不允许移除”，则：

    • 需要申请内存的指令(SET/LPUSH等)会返回错误信息

    • 无内存申请的指令(GET等)仍会正常响应

    样本数量
    maxmemory-samples 5

    注：数值越小越不准确，但性能消耗较小

    内存移除规则
    # volatile-lru -> 使用LRU算法移除key，只对设置了过期时间的键(少用)
# allkeys-lru -> 在所有集合key中，使用LRU算法移除key
# volatile-lfu -> LFU算法
# allkeys-lfu -> LFU算法
# volatile-random -> 在过期集合中移除随机的key，只对设置了过期时间的键
# allkeys-random -> 在所有集合中移除随机的key，只对设置了过期时间的键
# volatile-ttl -> 移除TTL最小(快要过期)的key
# noeviction -> 不进行移除
    Redis淘汰算法
    • 随机
    • TTL 过期时间
    • LRU（Least Recently Used，最近最少使用）
    • LFU（Least Frequently Used，最不经常使用）

    5.发布和订阅

    Redis发布订阅(pub/sub)是一种消息通信模式：发送者(pub)发送消息，订阅者(sub)接收消息

    Redis客户端可以订阅任意数量的频道

    测试

    客户端1：

    127.0.0.1:6379> publish channel1 hello
(integer) 1

    客户端2：

    127.0.0.1:6379> subscribe channel1
Reading messages... (press Ctrl-C to quit)
1) "subscribe"
2) "channel1"
3) (integer) 1
1) "message"
2) "channel1"
3) "hello"

    注：多个客户端订阅相同频道，都可以收到消息
