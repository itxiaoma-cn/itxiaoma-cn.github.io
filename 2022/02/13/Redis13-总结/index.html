    0.Docker连接Redis

    docker exec -it redis /bin/bash -c "redis-cli -c -p 6379 -a 123456"

    1.NoSql适用和不适用场景

    • 适用场景：高并发、海量数据读写，高可扩展性
    • 不适用场景：需要事务支持，结构化查询存储，处理复杂关系

    2.Memcache，Redis，MongoDB区别

    • Memcache：数据在内存中，不支持持久化，一般作为缓存数据库。Memcache是多线程 + 锁。
    • Redis：数据在内存中，支持持久化，支持多种数据结构，一般作为缓存数据库。Redis是单线程 + 多路 IO 复用。操作都是原子性的。
    • MongoDB：
      • 文档型数据库
      • 数据在内存中，内存不足时保存到硬盘
      • 支持二进制数据和大型对象
      • 配合/替代关系型数据库(RDBMS)存储特定数据

    3.Redis适用场景

    • 高频次，热门访问的数据，降低数据库 IO。
    • 分布式架构，做 session 共享。

    4.Redis数据类型及适用场景

    五大数据类型：string (字符串)、list (链表)、set (集合)、zset (sorted set –有序集合) 和 hash（哈希类型）。

    适用场景：

    • List：
      • 时间排序 => 最新N个数据
      • 构建队列
    • Zset：排行榜
    • Set：大量数据去重

    特性场景：

    • Expire过期时间：验证码过期时间
    • 原子性：计数器，秒杀（INCR、DECR）
    • PUBSUB：发布订阅消息系统

    5.Key常用命令

    查看当前库所有key

    keys *

    设置key值与value

    set key value

    判断key是否存在

    exists key

    查看key是什么类型

    type key

    删除指定的key数据

    del key

    设置过期时间

    expire key 10 //10秒

    查看还有多少秒过期

    ttl key //-1表示永不过期，-2表示已过期

    6.String类型数据结构及常用命令

    数据结构：简单动态字符串SDS

    扩容：字符串长度小于 1M 时，扩容都是加倍现有的空间，如果超过 1M，扩容时一次只会多扩 1M 的空间。

    最大值：一个字符串value最多可以是512M

    设置key值

    set key value

    查询key值

    get key

    删除key值

    del key

    将给定的value追加到原值末尾

    append key value

    获取值的长度

    strlen key

    有在key不存在的时候，设置key值

    setnx key value

    自增

    incr key

    自减

    decr key

    步长增减

    incrby/decrby key <步长>

    其他

    同时设置一个或者多个key-value                                                    mset key value key value...
同时获取一个或多个value                                                                  mget key key...
同时设置一个或者多个key-value.当且仅当所有给定key都不存在    msetnx key value key value... 
获取key的起始位置和结束位置的值(起始0)                                         getrange key <起始位置> <结束位置>
将value的值覆盖起始位置开始                                                             setrange key <起始位置> value     
设置键值的同时,设置过期时间                                                                setex key <时间> value    
用新值换旧值                                                                                        getset key value

    7.List类型数据结构及常用命令

    List由quicklist组成，是由多个压缩列表组成的双向链表，也就是将多个ziplist使用双向指针串起来使用。

    插入一个或者多个值

    lpush/rpush key value value...

    吐出一个或者多个值

    lpop/rpop key

    key1右边吐出一个值,插入到key2的左边

    rpoplpush key1 key2

    按照索引下标获取元素

    lrange key start stop

    获取所有值

    lrange key 0 -1

    按照索引下标获得元素

    lindex key index

    获取列表长度

    llen key

    在value的前面/后面插入一个新值

    linsert key before/after value newvalue

    从左边删除n个value值

    lrem key n value

    在列表key中的下标index中修改值value

    lset key index value //从0开始

    8.Set数据结构及常用命令

    Set数据结构是 dict 字典，字典底层是哈希表，所以添加，删除，查找的复杂度都是O(1)

    元素加入集合

    sadd key value value...

    取出该集合的所有值

    smembers key

    判断该集合key是否含有该值

    sismember key value

    返回该集合的元素个数

    scard key

    删除集合中的某个元素

    srem key value value...

    随机从集合中取出一个元素

    spop key

    随机从该集合中取出n个值（不会从集合中删除）

    srandmember key n

    一个集合a的某个value移动到另一个集合b

    smove a b value

    返回两个集合的交集元素

    sinter a b

    返回两个集合的并集元素

    sunion a b

    返回两个集合的差集元素（key1有的，key2没有）

    sdiff a b

    9.Hash数据结构及常用命令

    Hash是键值对集合，底层是string类型的field和value映射表，适合存储对象。

    Hash的底层存储有两种数据编码：ziplist（压缩列表），hashtable（哈希表）。

    ziplist编码条件：

    • hash对象保存的键和值字符串长度都小于64字节

    • hash对象保存的键值对数量小于512

    redis.conf

    hash-max-ziplist-value 64     #字符串长度都小于64字节
hash-max-ziplist-entries 512  #元素数量小于512

    给key集合中的filed键赋值value

    hset key field value

    集合field取出value

    hget key field

    批量设置hash的值

    hmset key field1 value1 field2 value2

    查看哈希表key中，给定域field是否存在

    hexist key1 field

    列出该hash集合的所有field

    hkeys key

    列出该hash集合的所有value

    hvals key

    为哈希表key中的域field的值加上增量1

    hincrby key field increment

    将哈希表key中的域field的值设置为value，当且仅当域field不存在

    hsetnx key field value

    10.Zset数据结构及常用命令

    zset 底层使用了两个数据结构：

    1. hash，hash 的作用就是关联元素 value 和权重 score，保障元素 value 的唯一性，可以通过元素 value 找到相应的 score 值。
    2. 跳跃表(skiplist)，跳跃表的目的在于给元素 value 排序，根据 score 的范围获取元素列表。

    加入Set

    zadd key score value

    根据下标返回元素

    zrange key start stop (withscores) //带withscores，可以让分数一起和值返回到结果集

    根据分数返回元素

    zrangebyscore key min max (withscores) //按score的值递增次序排列

    从大到小排列返回

    zrevrangebyscore key max min (withscores)

    为元素的score加上增量

    zincrby key increment value

    删除该集合下，指定值的元素

    zrem key value

    元素个数

    zcard key

    统计该集合，分数区间内的元素个数

    zcount key min max

    返回值在集合中的排名

    zrank key value //从0开始
